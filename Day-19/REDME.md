# 📌 Project Overview

On Day 19 of my AI Automation & Agentic AI Internship, I explored **Google Antigravity**, an AI-powered development environment that enables autonomous software development using intelligent agents.

The objective of this project was to recreate an automation workflow previously developed in **n8n** using an agent-first development approach. Instead of manually writing the entire application, I described the required functionality through a natural language prompt. Google Antigravity generated an implementation plan, created the Python application, executed it, and verified the final output automatically.

# 🔄 n8n vs Google Antigravity

This project involved recreating an automation that was previously built using **n8n**. Although both platforms automate business processes, they provide very different development experiences.

In **n8n**, workflows are created visually by connecting nodes representing triggers, conditions, AI services, and outputs. This approach is intuitive for automation and API integrations because every step is represented graphically. It is especially useful for users who prefer low-code or no-code development.

Google Antigravity follows a different approach. Instead of manually building workflows, I described the application requirements using a natural language prompt. The AI agent generated an implementation plan, created the Python application, executed it automatically, and verified the output. This significantly reduced manual coding while still producing clean and structured source code that can be modified later.

Unlike n8n, which focuses on visual automation, Google Antigravity emphasizes autonomous software development through AI agents. It enables developers to move from an idea to a working application more quickly while still allowing manual edits and verification.

Both platforms simplify automation, but each serves different purposes. n8n excels at workflow orchestration, whereas Google Antigravity is designed for AI-assisted software development.
